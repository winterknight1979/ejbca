108 /**
109  * API of available methods on the CA that can be invoked by the RA.
110  *
111  * <p>Implementation restrictions:
112  *
113  * <ul>
114  * <li> Keep in mind that there is latency, so batch things and don't for things twice unless it is expected to have change.
115  * <li> Method names must be unique and signature is not allowed change after a release
116  * <li> Any used object in this class must be Java Serializable
117  * <li> Any used object in this class should be possible to use with an older or newer version of the peer
118  * <li> Checked Exceptions are forwarded in full the implementation is responsible for not leaking sensitive information in
119  *   nested causedBy exceptions.
120  * <li> Query both local and remote, when applicable. Usually, local should be queried first for best performance.
121  * <li> Avoid having WS or REST-specific operations here (sometimes this is hard to avoid)
122  * <li> Try to put complex business logic into "business logic EJBs" such as CaSession etc. rather than here.
123  * <li> Remember to put @since attributes on new methods.
124  * </ul>
125  *
126  * <p>See the "RA Master API conventions" page in Confluence for more detailed information.
127  *
128  * @version $Id: RaMasterApi.java 29784 2018-08-30 08:20:30Z tarmo_r_helmes $
129  */
