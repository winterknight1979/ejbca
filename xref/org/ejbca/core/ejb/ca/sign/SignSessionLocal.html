50      /**
51       * Returns a CA that a request is targeted for. Uses different methods in priority order to try to find it.
52       * 
53       * @param admin an authenticating token
54       * @param req the request
55       * @param doLog if this operation should log in the audit log.
56       * @return CA object
57       * @throws CADoesntExistsException fail
58       * @throws AuthorizationDeniedException fail
59       */
60      CA getCAFromRequest(AuthenticationToken admin, RequestMessage req, boolean doLog) throws CADoesntExistsException, AuthorizationDeniedException;

62      /**
63       * Requests for a certificate to be created for the passed public key wrapped in a certification request message (ex PKCS10).  The username and password used to 
64       * authorize is taken from the request message. Verification of the signature (proof-of-possesion) on the request is performed, and an exception thrown if verification fails. 
65       * The method queries the user database for authorization of the user.
66       * 
67       * Works like the standard methods in this class for creating certificates, but will set status to NEW if it is GENERATED. Is guaranteed to roll back user status change if an 
68       * error is encountered during certificate creation. 
69       * 
70       *
71       * @param admin         Information about the administrator or admin performing the event.
72       * @param req           a Certification Request message, containing the public key to be put in the
73       *                      created certificate. Currently no additional parameters in requests are considered!
74       * @param responseClass The implementation class that will be used as the response message.
75       * @param ignorePassword set to true if password authentication isn't available (for example during SCEP certificate renewal). Password will be set from the request.
76       *  
77       * @return The newly created response
78       * 
79       * @throws NoSuchEndEntityException      if the user does not exist (does not require rollback)
80       * @throws CertificateRevokeException if certificate was meant to be issued revoked, but could not.  (rollback)
81       * @throws CertificateCreateException if certificate couldn't be created.  (rollback)
82       * @throws AuthorizationDeniedException if the authentication token wasn't authorized to the CA defined in the request  (rollback)
83       * @throws InvalidAlgorithmException fail
84       * @throws ApprovalException if changing the end entity status requires approval (does not require rollback)
85       * @throws WaitingForApprovalException if an approval is already waiting for the status to be changed (does not require rollback)
86       */
87      ResponseMessage createCertificateIgnoreStatus(final AuthenticationToken admin, final RequestMessage req,
88              Class<? extends CertificateResponseMessage> responseClass,  boolean ignorePassword) throws AuthorizationDeniedException, NoSuchEndEntityException,
89              CertificateCreateException, CertificateRevokeException, InvalidAlgorithmException, ApprovalException, WaitingForApprovalException;

91      /**
92       * Requests for a certificate to be created for the passed public key with default key usage
93       * The method queries the user database for authorization of the user.
94       *
95       * @param admin    Information about the administrator or admin performing the event.
96       * @param username unique username within the instance.
97       * @param password password for the user.
98       * @param pk       the public key to be put in the created certificate.
99       * @return The newly created certificate or null.
100      * 
101      * @throws AuthorizationDeniedException (rollback) if admin is not authorized to issue this certificate
102      * @throws CADoesntExistsException if the CA defined by caId doesn't exist.
103      * @throws AuthStatusException     If the users status is incorrect.
104      * @throws AuthLoginException      If the password is incorrect.
105      * @throws InvalidAlgorithmException if the signing algorithm in the certificate profile (or the CA Token if not found) was invalid.
106      * @throws CAOfflineException if the CA was offline
107      * @throws IllegalValidityException if the validity defined by notBefore and notAfter was invalid
108      * @throws CryptoTokenOfflineException if the crypto token for the CA wasn't found
109      * @throws CertificateSerialNumberException if certificate with same subject DN or key already exists for a user, if these limitations are enabled in CA.
110      * @throws CertificateRevokeException (rollback) if certificate was meant to be issued revoked, but could not.
111      * @throws IllegalNameException if the certificate request contained an illegal name 
112      * @throws CertificateCreateException (rollback) if certificate couldn't be created.
113      * @throws IllegalKeyException if the public key didn't conform to the constrains of the CA's certificate profile.
114      * @throws CustomCertificateSerialNumberException (no rollback) if custom serial number is registered for user, but it is not allowed to be used (either
115      *             missing unique index in database, or certificate profile does not allow it
116      * @throws NoSuchEndEntityException if the end entity was not found
117      */
118     Certificate createCertificate(AuthenticationToken admin, String username, String password, PublicKey pk) throws
119             CADoesntExistsException, AuthorizationDeniedException, IllegalKeyException, CertificateCreateException, IllegalNameException,
120             CertificateRevokeException, CertificateSerialNumberException, CryptoTokenOfflineException, IllegalValidityException, CAOfflineException,
121             InvalidAlgorithmException, CustomCertificateSerialNumberException, AuthStatusException, AuthLoginException, NoSuchEndEntityException;

123     /**
124      * Requests for a certificate to be created for the passed public key with the passed key
125      * usage. The method queries the user database for authorization of the user. CAs are only
126      * allowed to have certificateSign and CRLSign set.
127      *
128      * @param admin    Information about the administrator or admin performing the event.
129      * @param username unique username within the instance.
130      * @param password password for the user.
131      * @param pk       the public key to be put in the created certificate.
132      * @param keyusage integer with bit mask describing desired keys usage, overrides keyUsage from
133      *                 CertificateProfiles if allowed. Bit mask is packed in in integer using constants
134      *                 from CertificateData. -1 means use default keyUsage from CertificateProfile. ex. int
135      *                 keyusage = CertificateData.digitalSignature | CertificateData.nonRepudiation; gives
136      *                 digitalSignature and nonRepudiation. ex. int keyusage = CertificateData.keyCertSign
137      *                 | CertificateData.cRLSign; gives keyCertSign and cRLSign
138      * @param notBefore daye
139      * @param notAfter an optional validity to set in the created certificate, if the profile allows validity override, null if the profiles default validity should be used.
140      * @return The newly created certificate or null.
141      * 
142      * @throws NoSuchEndEntityException if the user does not exist.
143      * @throws AuthorizationDeniedException (rollback) if admin is not authorized to issue this certificate
144      * @throws CADoesntExistsException if the CA defined by caId doesn't exist.
145      * @throws AuthStatusException     If the users status is incorrect.
146      * @throws AuthLoginException      If the password is incorrect.
147      * @throws InvalidAlgorithmException if the signing algorithm in the certificate profile (or the CA Token if not found) was invalid.
148      * @throws CAOfflineException if the CA was offline
149      * @throws IllegalValidityException if the validity defined by notBefore and notAfter was invalid
150      * @throws CryptoTokenOfflineException if the crypto token for the CA wasn't found
151      * @throws CertificateSerialNumberException if certificate with same subject DN or key already exists for a user, if these limitations are enabled in CA.
152      * @throws CertificateRevokeException (rollback) if certificate was meant to be issued revoked, but could not.
153      * @throws IllegalNameException if the certificate request contained an illegal name 
154      * @throws CertificateCreateException (rollback) if certificate couldn't be created.
155      * @throws IllegalKeyException if the public key didn't conform to the constrains of the CA's certificate profile.
156      * @throws CustomCertificateSerialNumberException (no rollback) if custom serial number is registered for user, but it is not allowed to be used (either
157      *             missing unique index in database, or certificate profile does not allow it
158      *             
159      */
160     Certificate createCertificate(AuthenticationToken admin, String username, String password, PublicKey pk, int keyusage, Date notBefore,
161             Date notAfter) throws CADoesntExistsException, AuthorizationDeniedException, AuthStatusException,
162             AuthLoginException, IllegalKeyException, CertificateCreateException, IllegalNameException, CertificateRevokeException,
163             CertificateSerialNumberException, CryptoTokenOfflineException, IllegalValidityException, CAOfflineException, InvalidAlgorithmException,
164             CustomCertificateSerialNumberException, NoSuchEndEntityException;

166     /**
167      * Requests for a certificate to be created for the passed public key with the passed key
168      * usage and using the given certificate profile. This method is primarily intended to be used when
169      * issuing hardtokens having multiple certificates per user.
170      * The method queries the user database for authorization of the user. CAs are only
171      * allowed to have certificateSign and CRLSign set.
172      *
173      * @param admin                Information about the administrator or admin performing the event.
174      * @param username             unique username within the instance.
175      * @param password             password for the user.
176      * @param pk                   the public key to be put in the created certificate.
177      * @param keyusage             integer with bit mask describing desired keys usage, overrides keyUsage from
178      *                             CertificateProfiles if allowed. Bit mask is packed in in integer using constants
179      *                             from org.bouncycastle.jce.X509KeyUsage. -1 means use default keyUsage from CertificateProfile. ex. int
180      *                             keyusage = X509KeyUsage.digitalSignature | X509KeyUsage.nonRepudiation; gives
181      *                             digitalSignature and nonRepudiation. ex. int keyusage = X509KeyUsage.keyCertSign
182      *                             | X509KeyUsage.cRLSign; gives keyCertSign and cRLSign
183      * @param notBefore an optional validity to set in the created certificate, if the profile allows validity override, null if the profiles default validity should be used.
184      * @param notAfter an optional validity to set in the created certificate, if the profile allows validity override, null if the profiles default validity should be used.
185      * @param certificateprofileid used to override the one set in userdata.
186      *                             Should be set to CertificateProfileConstants.CERTPROFILE_NO_PROFILE if the usedata certificateprofileid should be used
187      * @param caid                 used to override the one set in userdata.
188      *                             Should be set to SecConst.CAID_USEUSERDEFINED if the regular certificateprofileid should be used
189      * 
190      * 
191      * @return The newly created certificate or null.
192      * 
193      * @throws AuthorizationDeniedException (rollback) if admin is not authorized to issue this certificate
194      * @throws CADoesntExistsException if the CA defined by caId doesn't exist.
195      * @throws AuthStatusException     If the users status is incorrect.
196      * @throws AuthLoginException      If the password is incorrect.
197      * @throws InvalidAlgorithmException if the signing algorithm in the certificate profile (or the CA Token if not found) was invalid.
198      * @throws CAOfflineException if the CA was offline
199      * @throws IllegalValidityException if the validity defined by notBefore and notAfter was invalid
200      * @throws CryptoTokenOfflineException if the crypto token for the CA wasn't found
201      * @throws CertificateSerialNumberException if certificate with same subject DN or key already exists for a user, if these limitations are enabled in CA.
202      * @throws CertificateRevokeException (rollback) if certificate was meant to be issued revoked, but could not.
203      * @throws IllegalNameException if the certificate request contained an illegal name 
204      * @throws CertificateCreateException (rollback) if certificate couldn't be created.
205      * @throws IllegalKeyException if the public key didn't conform to the constrains of the CA's certificate profile.
206      * @throws CustomCertificateSerialNumberException (no rollback) if custom serial number is registered for user, but it is not allowed to be used (either
207      *             missing unique index in database, or certificate profile does not allow it
208      * @throws NoSuchEndEntityException if the user does not exist.
209      * 
210      */
211      Certificate createCertificate(AuthenticationToken admin, String username, String password, PublicKey pk, int keyusage, Date notBefore,
212             Date notAfter, int certificateprofileid, int caid) throws CADoesntExistsException, AuthorizationDeniedException,
213             AuthStatusException, AuthLoginException, IllegalKeyException, CertificateCreateException, IllegalNameException,
214             CertificateRevokeException, CertificateSerialNumberException, CryptoTokenOfflineException, IllegalValidityException, CAOfflineException,
215             InvalidAlgorithmException, CustomCertificateSerialNumberException, NoSuchEndEntityException;

217      /**
218       * Signs the provided payload using the given CA's signing keys. 
219       * 
220       * @param data a byte array of the data to be signed
221       * @param signingCaId the name of the CA 
222       * @return an encoded CMSSignedData of the original payload
223       * @throws AuthorizationDeniedException if the authentication token was not authorized to the given CA
224       * @throws CryptoTokenOfflineException if the crypto token was offline 
225       * @throws CADoesntExistsException if a CA by the given name doesn't exist.
226       * @throws SignRequestSignatureException if an error occurred during the signature process. 
227       */
228      byte[] signPayload(byte[] data, final int signingCaId)
229              throws AuthorizationDeniedException, CryptoTokenOfflineException, CADoesntExistsException, SignRequestSignatureException;
