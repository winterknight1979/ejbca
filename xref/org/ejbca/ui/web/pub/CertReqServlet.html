39  /**
40   * Servlet used to install a private key with a corresponding certificate in a browser. A new
41   * certificate is installed in the browser in following steps:<br>
42   * 1. The key pair is generated by the browser. <br>
43   * 2. The public part is sent to the servlet in a POST together with user info ("pkcs10|keygen",
44   * "inst", "user", "password"). For internet explorer the public key is sent as a PKCS10
45   * certificate request. <br>
46   * 3. The new certificate is created by calling the RSASignSession session bean. <br>
47   * 4. A page containing the new certificate and a script that installs it is returned to the
48   * browser. <br>
49   * 
50   * <p>
51   * 
52   * <p>
53   * The following initiation parameters are needed by this servlet: <br>
54   * "responseTemplate" file that defines the response to the user (IE). It should have one line
55   * with the text "cert =". This line is replaced with the new certificate. "keyStorePass".
56   * Password needed to load the key-store. If this parameter is none existing it is assumed that no
57   * password is needed. The path could be absolute or relative.<br>
58   * </p>
59   *
60   * @version $Id: CertReqServlet.java 25830 2017-05-10 13:36:45Z mikekushner $
61   */
